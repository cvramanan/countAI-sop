# Ticket Closing Process SOP

---

## 1. Objective

To ensure all customer issues are properly documented, verified, and communicated before closing a support ticket.

---

## 2. Scope

This SOP applies to all tickets created for customer issues related to the **KNITI system**.

---

## 3. Responsibility

**Customer Support Executive is responsible for:**

- Updating ticket progress
- Documenting resolution details
- Confirming communication with customer/internal team
- Closing the ticket

---

## 4. Ticket Closing Procedure

Before closing any ticket, the following details must be updated in the tracking system:

---

### Step 1: Ticket Status Update

- Change ticket status: Open → In Progress → Completed

- 
---

### Step 2: Reference Evidence

Attach supporting evidence:

- Reference image of the issue
- Resolution image (after fix)

---

### Step 3: Resolution Details

The ticket must include:

- Problem description
- Temporary fix (if applied)
- Permanent fix
- Root cause of the issue

---

### Step 4: Communication Confirmation

Update whether resolution details were shared with:

- Customer  
**OR**
- Internal team  

---

### Step 5: Final Verification

Ensure the following fields are completed before closing:

- Ticket status updated
- Root cause mentioned
- Fix details documented
- Images attached
- Customer informed

Then mark the ticket as: Closed


---

## 5. Ticket Closing Checklist

| Item                            | Required            |
|----------------------------------|---------------------|
| Status updated                  | Yes                 |
| Reference image attached        | Yes                 |
| Temporary fix documented        | Yes (if applicable) |
| Permanent fix documented        | Yes                 |
| Root cause updated              | Yes                 |
| Resolution image attached       | Yes                 |
| Customer/internal team informed | Yes                 |
