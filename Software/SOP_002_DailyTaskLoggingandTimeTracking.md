# STANDARD OPERATING PROCEDURE (SOP)

**SOP ID:** SOP 003  
**Title:** Daily Task Logging and Time Tracking in Jira  
**Version:** 1.0  
**Effective Date:** 12.02.2026  
**Prepared By:** Siva  
**Approved By:** —  

---

## 1. Purpose

This SOP defines the standardized process for daily task logging, time tracking, and monitoring in Jira.

The objective is to:

- Ensure all developers log their daily work properly  
- Track planned vs unplanned tasks  
- Identify blockers early  
- Monitor extra hours spent  
- Improve task planning accuracy  
- Reduce time overrun  

---

## 2. Scope

This SOP applies to:

- Project Manager (PM)  
- All Developers  
- All active Jira projects  

---

## 3. Roles and Responsibilities

### 3.1 Developers

Developers are responsible for:

- Logging daily tasks in Jira  
- Updating task status correctly  
- Recording time spent  
- Adding proper comments  
- Logging additional/unplanned tasks  

### 3.2 Project Manager (PM)

The PM is responsible for:

- Reviewing daily assigned tasks after morning stand-up  
- Monitoring time logs in the evening  
- Identifying delays or extra effort  
- Tracking unplanned task count  
- Taking corrective actions if needed  

---

## 4. Procedure

### 4.1 Morning Stand-up (Before 11:00 AM)

During the stand-up meeting, each developer must share:

1. What was completed yesterday  
2. What is planned for today  
3. Any blockers  

After the meeting, the PM must:

- Verify the number of tasks assigned for the day  
- Ensure tasks are properly assigned in Jira  
- Confirm priorities  

---

### 4.2 Task Logging During the Day

Developers must:

- Work only on assigned tasks  
- Update task status (To Do → In Progress → Done)  
- Add meaningful comments explaining work done  
- Log additional tasks if any new work is assigned  

If any unplanned task arises:

- Create or log it in Jira  
- Mark it as unplanned  
- Add time spent clearly  

---

### 4.3 Evening Time Logging (5:00 PM – 6:00 PM)

Every developer must:

- Log actual time spent on each task  
- Add comments describing work completed  
- Update task status correctly  
- Ensure no task remains without time log  

Time log must include:

- Number of hours spent  
- Description of work done  
- Any challenges faced  

---

### 4.4 PM Review (After 5:30 PM)

The PM must:

- Verify all developers have logged time  
- Check planned vs actual hours  
- Identify extra hours spent  
- Identify blocked tasks  
- Review unplanned task count  

If issues are found:

- Discuss in next stand-up  
- Replan workload  
- Reduce unnecessary extra hours  

---

## 5. Monitoring Metrics

The following must be tracked weekly:

- Total planned tasks  
- Total completed tasks  
- Total unplanned tasks  
- Extra hours spent  
- Blocked task count  

---

## 6. Benefits

Following this SOP helps to:

- Improve task planning accuracy  
- Reduce time overrun  
- Identify bottlenecks early  
- Control unplanned work  
- Increase team accountability  
- Maintain proper project tracking  

---

## 7. Compliance

Failure to log time or update tasks daily may impact:

- Sprint tracking  
- Performance evaluation  
- Project delivery timelines  

All team members must strictly follow this SOP.
