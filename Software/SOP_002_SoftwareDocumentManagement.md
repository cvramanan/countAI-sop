# STANDARD OPERATING PROCEDURE (SOP)

## SOP ID: SOP_002 
## Title: Software Document Management and Storage Procedure  
## Version: 1.0  
## Effective Date: 12.02.2026  
## Prepared by: Siva  
## Approved by: --  

---

## 1. Purpose

This SOP defines the standardized process for creating, reviewing, approving, and storing all software-related documents in a centralized Google Drive location to ensure accessibility, security, version control, and traceability.

---

## 2. Scope

This procedure applies to:
- Software Project Manager (PM)
- Software Developers
- QA Team
- DevOps Team
- Stakeholders (view-only access where applicable)

---

## 3. Central Storage Location

All software team documents must be stored in:

> Official Software Team Google Drive (Central Repository)

No software-related document should be stored in personal drives or local systems without backup in the central repository.

---

## 4. Main Document Categories

The following documents must be maintained:

1. Software Release Notes  
2. Software User Manuals  
3. Important Images and Videos (Deployment, Testing, UI evidence)  
4. Account Credentials (Stored securely with restricted access)  
5. Software Asset Items (License details, tools, subscriptions)  
6. Site Links (Production, Staging, QA, Monitoring URLs)  
7. Technical Documents (Architecture, API Docs, DB Schema)  
8. SOPs and Process Documents  
9. Client Deliverables  
10. Backup and Archive Files  

---

## 5. Folder Structure Standard

The Software PM must maintain the following folder hierarchy inside the central Google Drive repository.

All documents must follow the structured path:

Software (Main Folder)  
→ Project Name Folder  
→ Related Document Category Folder  
→ Upload Versioned Document
